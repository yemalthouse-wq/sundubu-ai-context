# 2026-04-18 三銃士ログ

## What happened
- 気づきログ（GAS + スプレッドシート連携）が「動いているように見えるが記録されない」状態を確認
- fetch(no-cors)によりレスポンス検証不能な状態だった
- GAS doPostの受信確認が曖昧で、通信成否が判定できていなかった
- GitHub上で4/18のログファイル未作成のままworkflowが実行されエラー発生
- その状態のままVercelがデプロイを試み、preview deploymentが失敗

## So What
- 「UIが動く」と「データが記録される」は完全に別レイヤーであると認識
- ログ運用が習慣依存であり、システムとして担保されていなかった
- GitHub Actions（verify）はログ文化の強制装置として機能している
- 不整合な状態（ログ欠損）が下流（Vercelデプロイ）まで波及する構造が露呈
- 現在のログ基盤は「信頼できる状態」には未到達

## Now What
- GAS側にRECV_RAWシートを設け、通信受信を100%可視化
- GET → POSTの疎通確認フローを標準化
- fetch(no-cors)前提の設計を見直し、レスポンス検証可能な構成へ移行
- Queue & Retry（Local First）によるログ欠損ゼロ設計へ移行
- ログ未作成時に止まる現在のworkflowを前提とした運用ルールを徹底
