# Sundubu Tanaka OS

Sundubu Tanaka is not just a restaurant.

It is a reproducible urban lunch operating system.

---

## Core Idea

Most restaurants are built like this:

cooking
↓
store
↓
fatigue

Sundubu Tanaka is built like this:

market
↓
structure
↓
logistics
↓
replication

---

## System Layers

### 1. Market OS
Urban high-function lunch positioning

- lighter than ramen
- faster than set meals
- warm
- high protein

### 2. Kettle OS
Standardized production structure

- 1 kettle = 20 servings
- soup + mushroom mix + pork kimchi mix

### 3. Pricing OS
Hybrid pricing

- dine-in = brand experience
- delivery = revenue engine

### 4. Delivery OS
Platform-facing visual and pricing strategy

- red
- steam
- egg
- density

### 5. Supply Chain OS
FC is not a store model.
It is a logistics model.

- 1 box = 10 kettles = 200 servings

### 6. Franchise OS
Replication through

- kettle OS
- supply chain
- pricing model
- operating rules
