## 1. 設計原則（ぐぷちゃん提案）

### 原則① 3層分離

リポジトリのファイルを「読む頻度」で3層に分ける。

| 層 | 目的 | ファイル例 |
|---|---|---|
| **Layer 0：固定ルール** | 変わらない前提・確定値 | `core_rules.md`, `DECISION_LOG.md` |
| **Layer 1：現状スナップショット** | 今どこにいるか | `LATEST.md`, `ROADMAP.md` |
| **Layer 2：作業ログ** | 何をやったか | `logs/`, `docs/` |

Layer 0 を読まずにLayer 2 を読んではいけない。順番を守ることで、AIが毎回正しい前提で動ける。

### 原則② 書類を減らす

1ファイル1役割。複数の役割を1ファイルに詰め込まない。
迷ったら分割より削除を選ぶ。ファイルが増えるほど読む順番が崩れる。

### 原則③ 読む順番の固定

AIがどのセッションから入っても迷わないように、読む順番を固定する。

```
core_rules.md
→ DECISION_LOG.md
→ LATEST.md
→ ROADMAP.md
→ （タスクに応じて該当ファイル）
```

この順番は変更しない。CLAUDE.mdの冒頭に必ず明記する。

## 6. ゲンちゃんの追加提案

### 提案① GLOSSARY.md（用語集・定数セクション）

分析のたびに同じ定義を確認するコストを減らすため、用語集と定数を1ファイルに集約する。

**GLOSSARY.md の構成：**

```markdown
## 用語定義
- 1鍋：20食分のスンドゥブ素セット（固定）
- 1BOX：10鍋分（= 200食分）のセット
- 卸P/L：本部側の損益（BOX売価 − 商品原価 − 送料）
- 自店舗P/L：取引先側の損益（Uber売上 − 仕入れ代 − 店舗コスト）
- Go/No-Go：新丸子パイロットの継続判断（期限：2026年10月）

## 定数（2026-03-13更新）
- 1鍋原価：¥1,517〜¥1,522
- 1鍋卸値：¥3,500
- BOX価格：¥38,000（送料込・10鍋200食）
- 卸粗利：約¥19,780/BOX
- 卸単価（店内）：¥190/食
- 卸単価（デリ）：¥250/食（独立価格体系）
- FC加盟金：¥500,000（初回のみ）
- ロイヤリティ：¥0（供給マージン制）
- 物流費：¥30,000/店
```

AIがこのファイルを参照すれば、計算の前提がずれない。

### 提案② Log Rotation Rule

logsフォルダが増え続けると読みにくくなる。以下のルールで管理する。

| ルール | 内容 |
|---|---|
| **デイリーログ** | `logs/daily_log_YYYY-MM-DD.md` で毎日作成 |
| **月次アーカイブ** | 月末に `logs/archive/YYYY-MM/` へ移動 |
| **引き継ぎ文書** | `logs/handoff_*.md` は月次アーカイブに含める |
| **保持期限** | アーカイブは3ヶ月分を保持。それ以前は削除可 |
| **インデックス** | `logs/README.md` にログ一覧を記載（クロちゃんが管理） |

月次アーカイブの実施タイミング：毎月1日（クロちゃんが自動で実施）。

設計会議で合意した最終版。これ以降変更する場合はDECISION_LOGに記録する。

| 名前 | 正体 | 担当領域 | やらないこと |

### 役割越境の禁止

- クロちゃんが戦略的判断をしない（「〜すべきです」と言わない）
- ぐぷちゃんがコードやファイルを作成しない
- ゲンちゃんが意思決定をしない（数字を示すのみ）

判断はすべて**せいさん**が行う。三銃士は支援するだけ。
